# EdudoroIT_SupportCenter v1.0.0-rc1

Release Candidate 1.0 skupiony na domknięciu flow pracy IT Support Center: od zgłoszenia, przez weryfikację i realizację, po finalizację oraz transparentny podgląd obłożenia.

## Najważniejsze zmiany

### 1. Spójny flow zgłoszeń i zadań developerskich
- Ujednolicono przepływ między Ticketami, Kanban i TODO developera.
- Akceptacja i planowanie zgłoszeń automatycznie synchronizują statusy (`submitted`, `verified`, `in_progress`, `waiting`, `blocked`, `closed`).
- Powiązane taski developerskie są automatycznie tworzone/aktualizowane przy akceptacji i zmianach statusu.
- Reopen na Kanbanie przywraca poprawny status taska TODO (eliminuje rozjazdy między widokami).

### 2. Lista TODO developera (duży upgrade funkcjonalny)
- Jedna główna lista aktywna, scalająca pracę nad taskami i zgłoszeniami.
- Obsługa zadań przypisanych i nieprzypisanych w jednym miejscu.
- Lepsze filtrowanie i podsumowania (liczby, priorytety, estymacje).
- Start pracy (`in_progress`) i finalizacja przez modal z komentarzem.
- Zakończenie z wyborem ścieżki:
  - `closed` (zamknięte),
  - `waiting` (do weryfikacji przez zgłaszającego).
- Rozszerzony podgląd zadania z szybkim dostępem do kluczowych akcji.

### 3. Kanban i szczegóły zgłoszenia
- Podgląd zgłoszenia dostępny także na tablicy Kanban.
- W podglądzie i szczegółach wyeksponowano opis/treści zgłoszenia.
- Usprawniono edycję po stronie developera (status, plan, estymata, assignee).
- Dodano szybki przycisk akceptacji zgłoszenia w szczegółach ticketu.

### 4. Dashboard i obłożenie
- Dashboard zaktualizowany do najnowszego modelu statusów i metryk.
- Szybkie linki przebudowane na czytelne kafle (oddzielnie sekcja podstawowa i developerska).
- Sekcja alertów UX: stany OK/warning/critical i lepsza czytelność priorytetów.
- Widok Obłożenia pokazuje globalnie kolejki IT (niezależnie od zgłaszającego/developera) z informacją o przypisaniu.

### 5. Profil użytkownika i avatar
- Dodano panel profilu użytkownika:
  - edycja imienia i nazwiska,
  - email (read-only),
  - upload avatara.
- Avatar wyświetlany w UI (m.in. sidebar).
- Wejście do profilu przez klikalny blok użytkownika w stopce sidebara.

### 6. Admin i konfiguracja poczty
- Ustawienia aplikacji rozszerzone o upload logo i branding.
- Dodano wybór providera email:
  - SMTP
  - AWS SES
- Test wysyłki maila z panelu admina.
- Sekrety i dane wrażliwe są maskowane po stronie API.

### 7. UI/UX i design
- Duża przebudowa wizualna zgodnie z aktualną specyfikacją projektu.
- Lepsza responsywność (pełna szerokość contentu tam, gdzie potrzebna).
- Poprawki typografii, rozmiarów, czytelności i ekspozycji kluczowych elementów.
- Dodatkowe dopracowanie logowania i nawigacji sidebara.

## Backend / API

Nowe lub rozszerzone endpointy obejmują m.in.:
- profil i avatar użytkownika (`/api/auth/me`, `/api/auth/me/avatar`, `/api/auth/avatar/:filename`),
- publiczne branding settings (`/api/settings/public`, `/api/settings/logo`),
- konfigurację providerów email SMTP/SES oraz test maila (`/api/settings` + endpointy testowe),
- synchronizację flow ticket/task oraz workload (`/api/tickets/*`, `/api/dev-tasks/*`).

Usprawnienia:
- walidacje, RBAC i limity zapisu,
- bezpieczna obsługa uploadów i ścieżek plików,
- migracje kolumn użytkownika dla avatara wykonywane bezpiecznie przy starcie.
